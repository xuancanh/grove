# Grove Offline / Capacitor Architecture

> Review + design record (2026-07). Grove becomes a Capacitor app that works
> fully offline with **no login and no server**, while keeping the hosted
> server mode. The card/learning system adopts Knowledge Loom's data model,
> scheduler, and review UX.

## 1. Frontend ↔ server surface review

Every call the frontend makes lives in `src/lib/api.ts` (27 functions).
Offline strategy per group:

| Group | Endpoints | Offline strategy |
|---|---|---|
| status | `status` | constant `{ok, authRequired:false}` |
| catalog/books | `catalog`, `books/:id`, `books/:id/chapters/:no`, `POST books` (import), `DELETE books` | IndexedDB; public-domain catalog **bundled as JSON asset** (generated from grove-server's seed); text import runs the same splitter (`text-import.ts` ported to the client) |
| library | `library` CRUD, status/progress | IndexedDB |
| collections | create/delete/membership | IndexedDB |
| highlights | list/create/patch/delete | IndexedDB |
| cards | list/create/review/delete | IndexedDB + **FSRS-4.5** (see §3) |
| sessions | `POST sessions`, `sessions/rhythm` | IndexedDB; rhythm computed client-side |
| settings | get/patch | IndexedDB (single row) |
| AI | `ai/{status,chat,thread,cards,search}` | **direct-to-provider**: grove-server's `providers.ts` (plain fetch, no SDKs) is ported to the client; works when the user sets a provider+key in Settings. Needs network but **not our server**. On native, CapacitorHttp bypasses CORS; in the browser Anthropic (`anthropic-dangerous-direct-browser-access`) and Gemini allow direct calls, OpenAI generally does not — surfaced in Settings copy. |

## 2. Data layer

```
src/lib/data/
  provider.ts   DataProvider interface (the api.ts surface, typed)
  local.ts      LocalProvider — Dexie/IndexedDB, no auth, userId 'local'
  remote.ts     RemoteProvider — wraps the existing HTTP client
  index.ts      picks the provider: remote iff VITE_GROVE_SERVER is set,
                local otherwise (the Capacitor default)
```

Pages/components import `api` from `src/lib/data` — never from `api.ts`
directly. Local mode never shows the login screen.

## 3. Cards & learning — adopted from Knowledge Loom

Reviewed in the Loom repo: `server/src/scheduling/fsrs.ts` (FSRS-4.5,
default weights, desired retention 0.9, ratings again/hard/good → grades
1/2/3, `seedFromLegacy` for SM-2 rows), flashcard `reviewData` shape, and
the frontend study UX (`src/components/flashcards/`).

Grove changes to match:

- **Scheduler**: SM-2 → **FSRS-4.5** — the identical `fsrs.ts` file, used
  by the local provider and by grove-server (`POST cards/:id/review`).
  Existing SM-2 rows are migrated via `seedFromLegacy`.
- **Data model**: `Card` gains `reviewData { stability, difficulty, reps,
  lapses, intervalDays, nextReviewAt, lastReviewAt, lastRating }` (same
  field names as Loom). Ratings become `again | hard | good` with Loom's
  labels (Forgot / Tricky / Easy). `status` is derived, not stored.
- **Review UX** (replicated from Loom's `FlashcardStudy`/`FlashcardDone`):
  full-screen session with top bar (exit, progress track, `n / N`, live
  rating counts) · card **stack with ghost cards** and 3D flip · tap/Space
  to reveal · rating row with `1/2/3` keyboard shortcuts · swipe + arrow
  navigation · "previously rated" badge · stability/interval/reps chips ·
  next-review chip on the front · done screen with per-rating breakdown,
  mastered/needs-practice summary, and review-again.

## 4. Capacitor

`@capacitor/core` + `capacitor.config.ts` (`appId com.grove.reader`,
`webDir dist`); `ios/`/`android/` platforms generated by `npx cap add`.
The web build is unchanged — one codebase, three shells (web/PWA, iOS,
Android). Server mode still works by building with `VITE_GROVE_SERVER=1`
and hosting behind the enterprise backend.
